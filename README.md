# KV gRPC Service

gRPC сервис на Java для хранения пар ключ-значение с Tarantool в качестве базы данных.

## Стек

- Java 21+
- gRPC (grpc-java 1.76)
- Tarantool 3.2 (tarantool-java-sdk 1.5.0)
- Gradle 9
- Docker
- JUnit 5 + Testcontainers (тесты)

## API

| Метод | Описание |
|---|---|
| `Put(key, value)` | Сохраняет или перезаписывает значение по ключу. `value` опционален (может быть null) |
| `Get(key)` | Возвращает значение по ключу. Если ключ не найден — возвращает пустой ответ |
| `Delete(key)` | Удаляет запись по ключу |
| `Range(key_since, key_to)` | Возвращает stream пар ключ-значение в лексикографическом диапазоне включительно |
| `Count()` | Возвращает общее количество записей |

Схема данных в Tarantool (спейс `KV`):

```
key   : string       (primary key)
value : varbinary    (nullable)
```

## Запуск

### Требования

- Docker
- JDK 21+

### 1. Запустить Tarantool

```bash
docker-compose up -d
```

Tarantool поднимается на порту `3301`, создаёт спейс `KV` и пользователя `user`.

### 2. Настроить переменные окружения

Файл `.env` нужно создать в корне проекта, на основе `.env.exampl`

### 3. Запустить сервис

```bash
./gradlew run
```

Сервер стартует на порту `9090` и ждёт входящих gRPC запросов.

## Сборка

```bash
./gradlew build
```

## Тесты

Тесты используют Testcontainers — Docker должен быть запущен.

```bash
# все тесты
./gradlew test
```

Тесты производительности проверяют:
- корректность данных при наличии 1 000 000 записей
- время выполнения 1000 последовательных Get запросов (не более 10 секунд)
- время выполнения Range по 1000 записей (не более 3 секунд)
- корректный порядок результатов в Range

Количество записей для seed настраивается константой `SEED_COUNT` в `KvServicePerformanceTest.java`.
